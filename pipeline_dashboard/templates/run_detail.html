CDA (Continuous Double Auction) Energy Market

연속 양방향 경매 기반 전력 시장. 구매 입찰(Bid)과 판매 호가(Ask)가 Order Book에 등록되고, 매칭 엔진이 Highest Bid ≥ Lowest Ask 조건으로 체결하며 거래가는 (Bid+Ask)/2로 결정됩니다.

시장 흐름
Forecast / State → Strategy Agent (LLM) → Negotiation Layer → Policy / Trust → CDA Market → Settlement

시장 메커니즘

구성요소 설명
Order BookBid 테이블(가격 내림차순) · Ask 테이블(가격 오름차순)
MatchingHighest Bid ≥ Lowest Ask 일 때 체결, 거래가 = (Bid + Ask) / 2
BuyerDeficit / Market Price / Peak Risk 기반 구매 입찰 생성
Settlement거래 기록 반영, 정책 검증, Mesa 시뮬레이션 업데이트
