## 2. 현실 세계와의 대응 관계

### 2.1 그리드(grid) — 실제 배전계통

| 데이터 항목 | 현실 세계에서의 의미 |
|-------------|----------------------|
| **grid (141버스, 140브랜치, 1발전기)** | 카라카스(베네수엘라) 수도권 한 구역의 **12.47kV 방사형 배전계통** (MATPOWER case141) |
| **버스 1 + 발전기 1** | 변전소(수전점) — 상위 계통에서 전력을 공급받는 지점 |
| **나머지 140버스** | 피더 상의 수전·분기점(주상변압기, 인입점 등) |
| **140개 브랜치** | 중압 선로(가공/지중) 구간 — 저항·리액턴스로 모델링 |

**비교**: 토폴로지와 선로 파라미터는 **실제 계통에 기반**하지만, 테스트 pkl은 이 구조를 “고정된 배경”으로만 사용하고, **시계열(부하·태양광·가격 등)**은 아래와 같이 별도 생성됩니다.

### 2.2 프로슈머(prosumers) — 현실의 수용가·분산자원

20개 프로슈머는 **141버스 중 20개 버스**에 붙은 “소비+생산 가능한 수전점”으로, 현실의 **수용가 유형·설비 규모**를 타입별로 대표합니다.

| 타입 | 현실 세계 대응 | 설비 용량 (데이터) | 부하 규모(load_scale) |
|------|----------------|--------------------|------------------------|
| **Residential** | 일반 가정 | PV 6kW, BESS 13.5kWh/5kW, 부하 0.6 | 소규모 |
| **Commercial** | 상가·사무실 | PV 60kW, BESS 80kWh/30kW, CDG 40kW | 중간~중상 |
| **Industrial** | 공장·산업용 | WT 50kW, CDG 80kW, 가격반응 부하 30kW | 큼 |
| **Rural** | 농촌·농가 | PV 25kW, WT 35kW, BESS 50kWh/20kW | 중간 |
| **EnergyHub** | 마이크로그리드·복합단지 | PV 120kW, WT 60kW, BESS 200kWh/80kW, CDG 60kW | 가장 큼 |

**비교**:  
- **유형·용량 구간**은 현실의 소규모 주택(~수 kW)부터 중소 상가/공장(수십~100kW급)까지를 반영합니다.  
- 실제 계량 데이터가 아니라 **논문/시뮬레이션용 타입 스펙**으로 설계된 값입니다.

### 2.3 시계열(timeseries) — 현실과의 유사점·차이

각 행은 **(timestamp, prosumer_id)** 당 하나씩이며, 다음 컬럼을 가집니다.

| 컬럼 | 의미 | 현실 세계와의 비교 |
|------|------|--------------------|
| **load_kw** | 수요 [kW] | **유사**: 타입별 패턴(주거=저녁 피크, 상가=08~18시, 산업=06~22시 등)이 실제 수요 곡선과 유사. **차이**: 절대값은 ELIA 프록시·scale로 생성된 합성값. |
| **pv_kw** | 태양광 출력 [kW] | **유사**: 낮(약 06~18시)에만 발생, 정오 근처 최대 — 일사 패턴 반영. **차이**: 실제 일사·온도가 아니라 “낮/밤 + ELIA surplus” 기반 프록시. |
| **wt_kw** | 풍력 출력 [kW] | **유사**: 풍력이 있는 타입(Rural, Industrial, EnergyHub)만 0이 아님. **차이**: 실제 풍속이 아니라 ELIA 균형용량 기반 wind_proxy. |
| **bess_soc_kwh** | BESS 충전량 [kWh] | **유사**: 가격/일사에 따라 충·방전해 SOC가 변하는 패턴. **차이**: 단순 규칙(저가·고일사 충전, 고가 방전) 기반 시뮬레이션. |
| **price_buy / price_sell / price_p2p** | 전력 구매·판매·P2P 가격 | **유사**: 저녁 피크(17~21시)에 비싸고, 기본값+수급 긴장도 반영. **차이**: 실제 한국 전력시장 가격이 아니라 95~205원/kWh 구간의 합성 가격. |
| **controllable_load_kw** | 가격 반응형 부하 | **유사**: 고가 시각에 더 많이 쓰는 “수요반응” 형태. **차이**: 고가 구간 비율로 단순화된 규칙. |

**실제 데이터 통계(테스트 기간 전체)**  
- load_kw: 평균 약 34.5 kW, 범위 약 7.3~173.5 kW  
- pv_kw: 평균 약 6.6 kW, 최대 약 93.2 kW  
- wt_kw: 평균 약 10.5 kW, 최대 약 42.8 kW  
- price_buy: 약 95~205 원/kWh, 평균 약 101.8  
- bess_soc_kwh: 0~200 kWh (EnergyHub 200kWh 한도 반영)

## 3. 데이터 생성 방식과 현실성

### 3.1 입력 데이터의 출처

- **ELIA 원시 데이터**: 벨기에 전력계통운영자(ELIA)의 **균형보조용량(aFRR/mFRR)** 활성화량(스크린샷 스키마).  
- **테스트 기간**: 2026년 5월이 원본 ELIA에 없으므로, **2025년 5월 ELIA 데이터를 1년 이동**해 2026년 5월로 사용했습니다.

따라서 **시간대(서울)**와 **일/시간별 패턴**은 있지만, **한국 전력수급·한국 일사·한국 가격**이 아닙니다.

### 3.2 프록시(proxy) 사용

- **solar_proxy**: “낮/밤” + ELIA surplus 정규값으로 일사 유사 신호 생성.  
- **wind_proxy**: surplus/scarcity로 풍력 유사 신호 생성.  
- **load_proxy**: 저녁 피크 + scarcity로 수요 유사 신호 생성.  
- **price**: `95 + 90×scarcity + 20×(17~21시)` 등으로 합성.

→ **현실**: 부하·재생에너지·가격의 **형태(패턴)**는 현실과 비슷하게 맞춰져 있음.  
→ **한계**: 실제 계측·시장 데이터가 아니므로, 정량적 검증(예: 한국 5월 일사·KPX 가격과의 상관)에는 사용할 수 없음.

### 3.3 시간대(Asia/Seoul)

- 모든 **timestamp**는 **Asia/Seoul** 기준입니다.  
- 따라서 “오전 9시”, “저녁 6시” 등은 **한국 표준시**로 해석하면 됩니다.  
- 일사·부하 패턴도 이 시간대 기준으로 만들어져, **한국 기준 주·야 구간**과 대체로 맞습니다.

## 4. 현실 세계와 비교한 요약

| 구분 | 현실 세계와 맞는 부분 | 현실 세계와 다른 부분 |
|------|------------------------|------------------------|
| **계통 구조** | 141버스 배전계통은 실제 계통(case141) 기반, 12.47kV·방사형 | 한 구역만 표현, 상위계통·보강설비 등은 미모델링 |
| **프로슈머 유형** | 주거/상가/산업/농촌/허브 등 현실 유형·용량 구간 반영 | 실제 계량 데이터가 아닌 타입 스펙 |
| **부하 패턴** | 타입별 일일 패턴(저녁 피크, 업무시간대 등) 현실과 유사 | 절대값·일별 변동은 ELIA 프록시 기반 합성 |
| **태양광** | 낮에만 발생, 정오 근처 최대 — 일사 형태 유사 | 실제 일사·기상이 아닌 daylight+surplus 프록시 |
| **풍력** | 타입별 용량만 반영, 시간대별 변동 존재 | 실제 풍속이 아닌 ELIA 기반 wind_proxy |
| **가격** | 저녁 피크·수급 긴장 시 상승 — 형태는 현실과 유사 | 한국 전력시장 가격이 아닌 95~205 원 구간 합성 |
| **시간대** | 서울 시간 기준으로 해석 가능 | 원본은 유럽(ELIA) 데이터를 날짜만 2026년 5월로 이동 |

## 5. 테스트 데이터로 쓰는 관점

- **알고리즘·강화학습·시뮬레이션 검증**에는 적합:  
  - 그리드 구조는 실제에 가깝고,  
  - 부하·PV·풍력·가격의 **시간적 형태**가 현실과 비슷하며,  
  - 2026년 5월 1개월·서울 시간·20개 프로슈머가 명확히 구분됩니다.  

- **실제 계측값·한국 시장가와의 정량 비교**에는 사용하지 않는 것이 좋습니다.  
  - ELIA 균형보조 데이터와 프록시로 만든 합성 데이터이기 때문입니다.

이 파일을 기준으로 **학습된 모델의 테스트**나 **시나리오 분석**을 이어가면 됩니다.
