# LLM 프롬프트 (통합 파일)

에이전트에서 사용하는 **시스템 프롬프트**와 **유저 프롬프트**를 **한 파일** `prompts.txt` 에서 관리합니다.  
소스 수정 없이 문구만 바꾸려면 이 파일만 편집하면 됩니다.

## 파일

- **`prompts.txt`** — 모든 에이전트의 system/user 프롬프트가 하나에 들어 있습니다.

## 형식

- **섹션 구분**: `[에이전트명.system]` / `[에이전트명.user]` 한 줄로 구분합니다.
- **순서**: `forecast_planner.system` → `forecast_planner.user` → `decision.system` → `decision.user` → `validation.system` → `validation.user` (순서는 바꿔도 동작합니다).
- **인코딩**: UTF-8로 저장하세요.
- **주석**: `#` 으로 시작하는 줄은 파싱 시 무시됩니다(현재는 파일 상단 설명용).

## 수정 시 주의

- **유저 템플릿 placeholder**: `{prosumer_id}`, `{load_mean:.2f}` 등은 코드에서 `.format(**data)` 로 채우는 변수이므로, 이름을 바꾸거나 삭제하면 에러가 납니다. 문구만 바꾸거나 항목을 추가하는 것은 괜찮습니다.
- **시스템 프롬프트**: JSON 스키마(필드명)를 바꾸면 에이전트에서 파싱하는 키와 맞지 않을 수 있으니, 설명 문구만 수정하는 것을 권장합니다.
- **파일을 삭제하면** 로더 내장 기본값이 사용됩니다.

## 반영 시점

프롬프트는 **호출 시마다** 파일에서 읽습니다. 저장 후 다음 LLM 호출부터 반영됩니다.
